# نظام إدارة مزارع الدواجن متعددة العنابر (Poultry Management System)

## نظرة عامة
نظام ويب متكامل لإدارة مزارع الدواجن بعدة عنابر، مع تتبع الدورات (cycles)، المخزون، والحسابات المالية.

## المميزات الرئيسية

### 1. إدارة العنابر (Sheds Management)
- إنشاء وتعديل وحذف العنابر
- تتبع الحالة (نشط/غير نشط)
- عرض الدورات النشطة والمغلقة

### 2. إدارة الدورات (Cycles Management)
- إنشاء دورات جديدة لكل عنبر
- تتبع الكتاكيت الأولي
- تسجيل الوفيات (النافق)
- إغلاق الدورة مع كشف الفروقات

### 3. تتبع المخزون (Inventory Tracking)
- حساب الكتاكيت المتبقية تلقائياً: `الأولي - النافق`
- عرض إحصائيات مباشرة

### 4. الإدارة المالية (Financial Management)
- تسجيل المصروفات والإيرادات لكل دورة
- حساب صافي الحاصل تلقائياً
- حساب هامش الربح

### 5. التقارير السنوية (Annual Reports)
- عرض تقارير مالية شاملة
- مقارنة أداء الدورات
- طباعة التقارير

## متطلبات التثبيت
- PHP 8.2+
- Laravel 12
- MySQL/SQLite

## خطوات التشغيل

### 1. تثبيت المشروع
```bash
composer install
```

### 2. إنشاء ملف `.env`
```bash
cp .env.example .env
php artisan key:generate
```

### 3. إعداد قاعدة البيانات
في ملف `.env`:
```
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=poultry_farm
DB_USERNAME=root
DB_PASSWORD=
```

### 4. تشغيل الـ Migrations
```bash
php artisan migrate
```

### 5. تشغيل السيرفر
```bash
php artisan serve
```

الموقع متاح على: `http://localhost:8000`

## هيكل قاعدة البيانات

### جدول Sheds (العنابر)
| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | INT | المعرف الفريد |
| name | STRING | اسم العنبر |
| description | TEXT | الوصف |
| status | ENUM | الحالة (active/inactive) |
| timestamps | - | created_at, updated_at |

### جدول Cycles (الدورات)
| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | INT | المعرف الفريد |
| shed_id | INT | معرف العنبر (FK) |
| start_date | DATE | تاريخ البداية |
| end_date | DATE | تاريخ النهاية |
| initial_chicks | INT | عدد الكتاكيت الأولي |
| mortality_count | INT | عدد الوفيات |
| sold_chicks | INT | عدد المبيعات |
| status | ENUM | الحالة (active/completed) |
| timestamps | - | created_at, updated_at |

### جدول FinancialRecords (السجلات المالية)
| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | INT | المعرف الفريد |
| cycle_id | INT | معرف الدورة (FK) |
| type | ENUM | النوع (expense/revenue) |
| amount | DECIMAL | المبلغ |
| description | TEXT | الوصف |
| record_date | DATE | تاريخ السجل |
| timestamps | - | created_at, updated_at |

## العمليات الأساسية

### 1. إضافة عنبر جديد
```
الصفحة الرئيسية → إضافة عنبر جديد
```

### 2. إنشاء دورة جديدة
```
عنبر → الدورات النشطة → إنشاء دورة جديدة
```

### 3. تسجيل النافق
```
الدورة → إضافة نافق
```

### 4. إضافة سجل مالي
```
الدورة → السجلات المالية → إضافة سجل
```

### 5. إغلاق الدورة
```
الدورة → إغلاق الدورة (يتم كشف الفروقات تلقائياً)
```

### 6. عرض التقرير السنوي
```
السجلات المالية → التقرير السنوي
```

## الحسابات المدمجة

### الكتاكيت المتبقية
```
المتبقي = الأولي - النافق
```

### المصروفات الإجمالية
```
المصروفات = مجموع جميع السجلات من نوع "مصروف"
```

### الإيرادات الإجمالية
```
الإيرادات = مجموع جميع السجلات من نوع "إيراد"
```

### صافي الحاصل
```
صافي الحاصل = الإيرادات - المصروفات
```

### هامش الربح
```
الهامش = (صافي الحاصل ÷ الإيرادات) × 100
```

### كشف الفروقات
```
الفرق = الكتاكيت المتبقي - المباع
```

## التنسيقات المستخدمة

### اللغة
- واجهة كاملة باللغة العربية
- دعم RTL (اليمين لليسار)

### التواريخ
- الصيغة: `YYYY-MM-DD`
- الحفظ في UTC

### العملات
- عرض بـ 2 منزلة عشرية
- فاصل آلاف بـ `,`

## أدوار ومسؤوليات

### المتحكمات (Controllers)
1. **ShedController**: إدارة العنابر (CRUD)
2. **CycleController**: إدارة الدورات والعمليات الخاصة بها
3. **FinancialRecordController**: إدارة السجلات المالية والتقارير

### النماذج (Models)
1. **Shed**: علاقة One-to-Many مع Cycles
2. **Cycle**: علاقة Many-to-One مع Shed، One-to-Many مع FinancialRecords
3. **FinancialRecord**: علاقة Many-to-One مع Cycle

## قوائم الطلب (Routes)

### العنابر
```
GET    /sheds               - عرض جميع العنابر
GET    /sheds/create        - نموذج إنشاء عنبر
POST   /sheds               - حفظ عنبر جديد
GET    /sheds/{shed}        - عرض تفاصيل عنبر
GET    /sheds/{shed}/edit   - نموذج تعديل عنبر
PATCH  /sheds/{shed}        - تحديث بيانات عنبر
DELETE /sheds/{shed}        - حذف عنبر
```

### الدورات
```
GET    /cycles                      - عرض جميع الدورات
GET    /cycles/create               - نموذج إنشاء دورة
POST   /cycles                      - حفظ دورة جديدة
GET    /cycles/{cycle}              - عرض تفاصيل الدورة
GET    /cycles/{cycle}/mortality/edit   - نموذج إضافة نافق
PATCH  /cycles/{cycle}/mortality        - حفظ إضافة نافق
GET    /cycles/{cycle}/close        - نموذج إغلاق الدورة
PATCH  /cycles/{cycle}/close        - إغلاق الدورة
```

### السجلات المالية
```
GET    /cycles/{cycle}/financial-records/create     - نموذج إضافة سجل
POST   /cycles/{cycle}/financial-records              - حفظ سجل جديد
GET    /financial-records/{record}/edit              - نموذج تعديل سجل
PATCH  /financial-records/{record}                   - تحديث سجل
DELETE /financial-records/{record}                   - حذف سجل
GET    /financial-records/annual-report              - التقرير السنوي
```

## ملاحظات مهمة

1. **لا يمكن تعديل الدورات المغلقة**: بعد إغلاق دورة، لا يمكن تعديل سجلاتها المالية أو إضافة نافق جديد

2. **كشف الفروقات**: عند إغلاق دورة، يتم مقارنة الكتاكيت المتبقي بالمباع وتظهر رسالة توضح أي فروقات

3. **البيانات المحذوفة**: حذف عنبر سيؤدي لحذف جميع دوراته وسجلاته المالية

4. **التقارير**: يمكن طباعة التقارير السنوية مباشرة من المتصفح
